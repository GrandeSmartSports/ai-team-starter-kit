**Title:** My Story: From Zero Technical Knowledge to 8 AI Agents in 7 Weeks

**Body:**

Seven weeks ago I couldn't run a script in the terminal.

I'm not exaggerating. I didn't know what chmod meant. I didn't know what a path was. I had never opened Terminal on my Mac on purpose.

Let me back up.

My name is Daniel. I live in Bonney Lake, Washington. I work full time. I coach youth softball and soccer. I have a wife and two daughters, Lucy and Lily. My schedule is insane. I'm at practice three nights a week, games on weekends, and trying to build something on the side that could change my family's future.

I'm not a developer. I've never been a developer. I don't have a CS degree. I don't know Python or Swift or JavaScript. Not really.

But I had an idea for a product. CoachPilot. An AI-powered coaching platform that helps youth sports coaches with player evaluations, practice plans, game day lineups. All the stuff coaches struggle with but nobody builds tools for.

I just didn't know how to build it.

Then I found Claude Code.

It's an AI coding agent you run in your terminal. You tell it what to build. It writes the code. It runs the commands. It deploys the app. You direct. It builds.

My first project was embarrassingly simple. A single HTML file. A softball evaluation tool where coaches could rate players on a scale. That's it. One file. One page.

But it worked. And something clicked.

I didn't need to learn to code. I needed to learn to direct.

So I kept going. I told Claude to build a practice plans app. Then a walkup music app called OnDeck. Then a reading app for my daughter Lily called ReadQuest. Then the full CoachPilot platform in Swift for iOS.

One agent became two. I needed a builder to write code and a tester to check the work. Two became four. I added a designer and a team lead. Four became eight. Two full sub-teams, each with a lead, a builder, and a QA tester. Plus two independent agents for special projects.

I named them. Relay. Cortex. Bolt. Keeper. Forge. Patch. Axon. Reflex. Real names. Real roles. Real responsibilities.

And then everything broke.

The problems came fast.

My agents would forget who they were. After 15 or 20 messages, my brain architect would start acting like a generic coding assistant. My QA tester would start writing features instead of testing them. They'd drift.

They had no memory. Every new session started from zero. I'd spend 20 minutes re-explaining what we built yesterday before any new work could happen.

Two agents would edit the same file at the same time. One would overwrite the other's work. Nobody knew who was doing what.

And the scary one. An agent almost deleted a database. Another pushed code to production without asking. There were no guardrails. No boundaries. No rules.

So I built them. One at a time. Solving real problems as they hit me.

Identity anchoring came first. A file each agent reads at the start of every session that says "YOU ARE Bolt. You are NOT Relay. You are NOT Cortex." With a built-in check every 10 messages. Identity drift solved.

Then persistent memory. Each agent has their own memory file on disk. They read it at session start. They update it during work. Next session, they pick up where they left off. No more re-explaining.

Then security rules. Explicit boundaries for each agent. What they can do. What they cannot do. Anti-tunnel rules that prevent creative workarounds. Hard limits on production access, credential access, file deletion. Safety solved.

Then communication. A file-based inbox/outbox system. I drop work in an agent's inbox folder. They deliver results to their outbox folder. A mailroom script watches for new files and sends me a notification. Clean handoffs. No stepping on each other.

Then a Source of Truth. One document. One file. Every agent reads it. Current status, blockers, assignments, priorities. Everyone aligned. Always.

None of this was planned. I didn't sit down and design a system. I hit a wall, figured out a fix, wrote it down, and moved on. Over and over. For seven weeks.

Here's where I am now.

8 coordinated AI agents. Two sub-teams with leads.

CoachPilot for iOS. In TestFlight. Real coaches testing it.

OnDeck. A walkup music app. Deployed and working.

ReadQuest. A reading app I built for my daughter. She uses it.

Multiple web apps in production. Landing pages, community tools, internal dashboards.

I ship something almost every day. Not because I'm working 18-hour days. Because my team works while I direct. I spend maybe 45 minutes a week actually managing the agents. The rest of my "building" time is product decisions, testing, and giving feedback.

I built all of this in 7 weeks. Starting from zero.

Here's what I believe now.

This is the future. Not coding. Directing. Non-technical people running teams of AI agents to build real products. You don't need a CS degree. You don't need 10 years of engineering experience. You need clear thinking. You need to know what you want built. And you need a system that keeps your agents organized, focused, and safe.

That's what I packaged into the AI Team Starter Kit. My exact system. Identity anchoring, persistent memory, security rules, communication infrastructure, automation scripts. One setup script. Five questions. Your team is running in 10 minutes.

I built this community to share everything. How I work with my agents. What breaks. What I learn. Real behind-the-scenes from real products, not theory.

If you're here, you're early. This space is brand new. Most people are still using AI as a chatbot. A fancy autocomplete. You're about to use it as a team.

Ask me anything. I'm here. I respond to everything.

Let's build.

-- Daniel
